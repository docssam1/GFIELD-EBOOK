스크롤 타깃 구간 맵:

- 강의 버튼에 연결된 `scrollTarget` 값을 화면 아래 별도 구간 맵으로 렌더링
- 실제 교재에 id를 심기 전에도 데모 단계에서 스크롤 반응 확인 가능

## 4. 목록은 어디에 있는가

### 승인 학생 목록

위치:

- 관리자 파일 `gfield-on-archive-admin.html`
- 화면 기준 오른쪽 패널의 `승인 학생 목록` 영역

관리 방식:

- 한 줄에 한 명씩 입력
- `불러오기 / 수정 반영 / 저장하기` 버튼 사용

실제 저장:

- 브라우저 `localStorage`
- 키: `gfield_archive_admin_config`

저장 데이터 안에서는 `approvedStudents` 배열로 들어감

### 강의 버튼 목록

위치:

- 관리자 파일 `gfield-on-archive-admin.html`
- `강의 버튼 관리` 영역

실제 저장:

- 브라우저 `localStorage`
- 키: `gfield_archive_admin_config`

저장 데이터 안에서는 `lectureButtons` 배열로 들어감

### 상위 버튼 목록

위치:

- 관리자 파일 `gfield-on-archive-admin.html`
- `상위 버튼 관리` 영역

실제 저장:

- 브라우저 `localStorage`
- 키: `gfield_archive_admin_config`

저장 데이터 안에서는 `topLevelGroups` 배열로 들어감

## 5. 현재 동작 기준

1. 학생 이름 입력
2. 승인 학생 목록이 비어 있지 않으면, 목록에 있는 이름만 통과
3. 통과 후 과정명과 학생 이름 세션 저장
4. 미리보기 화면 진입
5. 상위 버튼 선택
6. 페이지 탭 선택
7. 강의 카드 클릭
8. 링크 열기 + 스크롤 타깃 이동

## 6. 아직 남은 핵심

- 실제 교재 내부에 `id` 또는 `data-scroll-target` 심기
- 버튼 클릭과 오른쪽 교재 위치를 정확히 맞추기
- 영상 변경과 교재 스크롤을 완전히 묶기

## 8. G.FIELD ON 플랫폼 이미지 핵심

이 항목은 앞으로의 화면 수정, 문구 수정, 이미지 선택, 카드 구성의 기준입니다.

### 1. 공개 강의몰처럼 보이면 안 됨

- 누구나 들어오는 사이트 느낌 금지
- 할인 / 홍보 / 과한 CTA 금지
- `재원생 전용`, `승인형`, `조용히 제공되는 서재` 느낌 유지

### 2. 화면이 곧 브랜드여야 함

- 첫 화면부터 고급스럽고 단정한 인상
- 화려함보다 품격
- 영상 / 이미지 / 문구가 한 방향이어야 함

### 3. 말투가 중요함

- 과장된 확신 금지
- 너무 상업적인 홍보 문구 금지
- 잔잔한 신뢰
- 조용한 자신감
- 준비된 느낌

예시 문구:

- `재원생에게 필요한 일부 영상강의와 자료를 조용히 제공하는 공간`
- `상담과 승인을 거쳐 입장할 수 있는 시크릿 서재`

### 4. 자료 수준이 플랫폼 수준을 결정함

- 교재 이미지
- 성적표 예시
- 학습 리포트
- 실제 영상 썸네일

핵심 기준:

- `아, 여긴 진짜 자료가 있는 곳이구나`가 보여야 함
- 빈 카드, 더미 느낌, 허전한 화면 금지

### 5. 접근 방식도 이미지다

- 이름 입력
- 승인된 학생만 입장
- 입소문으로만 전달되는 일부 서비스

이 접근 방식 자체가 프리미엄 이미지를 만든다.

### 6. 일관성이 가장 중요함

아래 요소는 모두 한 톤으로 유지:

- 이름
- 문구
- 색감
- 이미지
- 카드 스타일
- 영상 썸네일
- 안내 방식

## 9. 앞으로의 수정 원칙

앞으로 화면을 수정할 때는 다음을 항상 먼저 확인한다.

1. 공개 강의몰처럼 보이지 않는가
2. 빈 더미 카드처럼 보이지 않는가
3. 말투가 과장되거나 상업적으로 튀지 않는가
4. 이미지와 문구와 카드가 한 톤인가
5. 재원생 전용 / 승인형 / 시크릿 서재의 결이 유지되는가
